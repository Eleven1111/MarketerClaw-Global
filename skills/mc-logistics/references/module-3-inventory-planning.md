## Module 3: 库存管理与备货计划

### 3.1 需求预测框架

**基础预测公式：**

```
预测月销量 = 基准月销量 × 季节性系数 × 增长系数 × 广告系数 × 促销系数
```

| 系数 | 说明 | 数据来源 |
|------|------|---------|
| 基准月销量 | 近 3 个月日均销量 × 30 | 卖家后台 Business Report |
| 季节性系数 | 同比月份的销量波动比 | 去年同期数据 / 品类趋势 |
| 增长系数 | 环比增长趋势 | 近 6 个月增长率 |
| 广告系数 | 广告预算变化的销量影响 | 广告投放计划 |
| 促销系数 | 大促期间销量放大倍数 | 历史大促数据 / 品类基准 |

**季节性系数参考（Amazon US 通用品类）：**

| 月份 | 系数 | 说明 |
|------|------|------|
| 1月 | 0.7 | 节后淡季 |
| 2月 | 0.75 | 情人节小高峰 |
| 3月 | 0.85 | 春季回暖 |
| 4月 | 0.9 | 平稳期 |
| 5月 | 0.95 | 母亲节 + 临近夏季 |
| 6月 | 1.0 | 基准月（父亲节） |
| 7月 | 1.3 | Prime Day 大促 |
| 8月 | 1.0 | 返校季 |
| 9月 | 0.9 | 平稳期 |
| 10月 | 1.1 | 万圣节 + BFCM 预热 |
| 11月 | 1.5 | BFCM（黑五网一） |
| 12月 | 1.4 | 圣诞季 |

> 注意：以上为通用参考值，不同品类差异巨大。户外运动夏季系数更高，保暖品类冬季系数更高。

**促销系数参考：**

| 促销活动 | 通常放大倍数 |
|---------|------------|
| Lightning Deal | 2-5x |
| Best Deal | 1.5-3x |
| Prime Day（参加 Deal） | 5-15x |
| BFCM（参加 Deal） | 5-20x |
| Coupon（20% off） | 1.3-2x |

### 3.2 安全库存计算

**安全库存公式：**

```
安全库存 = Z × √(LT × σ_d² + d² × σ_LT²)

其中：
Z     = 服务水平对应的 Z 值（95% → 1.65，99% → 2.33）
LT    = 平均补货前置时间（天）
σ_d   = 日均销量标准差
d     = 日均销量均值
σ_LT  = 补货前置时间标准差（天）
```

**简化版（适合大多数卖家）：**

```
安全库存 = 日均销量 × 缓冲天数

缓冲天数建议：
├── 海运补货：14-21 天
├── 空运补货：7-10 天
├── 3PL 转 FBA：3-5 天
└── 旺季额外加 50%
```

### 3.3 补货点（Reorder Point）计算

**补货点公式：**

```
补货点 = 日均销量 × 补货前置时间 + 安全库存
```

**完整示例：**

```
产品：无线蓝牙耳机
日均销量：30 件
补货前置时间：45 天（海运 30 天 + 清关 5 天 + FBA 入库 10 天）
安全库存：30 × 14 = 420 件

补货点 = 30 × 45 + 420 = 1,770 件

解读：当 FBA 库存降至 1,770 件时，触发补货。
每次补货量 = 日均销量 × 补货周期 = 30 × 60（两个月） = 1,800 件
```

**FBA 可用库存天数：**

```
可用天数 = (FBA 当前可售数量 + 在途数量) ÷ 日均销量

健康区间：30-90 天
├── < 28 天 → 触发低库存费，紧急补货
├── 28-60 天 → 正常，启动下一批补货
├── 60-90 天 → 充足，观察
└── > 90 天 → 过多，考虑减速或促销清库存
```

### 3.4 Amazon IPI（Inventory Performance Index）优化

IPI 是 Amazon 对卖家库存管理能力的综合评分（0-1000）。

**IPI 评分因素：**

| 因素 | 权重 | 优化目标 |
|------|------|---------|
| **超龄库存占比** | 高 | < 25%（90 天以上库龄占比） |
| **售罄率（Sell-through Rate）** | 高 | > 7（近 90 天销量 ÷ 平均库存） |
| **FBA 在库率** | 中 | > 90%（畅销品不断货） |
| **滞销库存** | 中 | 及时清理 |

**IPI 分数区间与影响：**

| IPI 分数 | 状态 | 影响 |
|---------|------|------|
| > 550 | 优秀 | 无限制 |
| 400-550 | 良好 | 无限制 |
| 350-400 | 警告 | 可能被限制库容 |
| < 350 | 危险 | 严格库容限制，无法补货 |

**IPI 提升行动计划：**

1. **清理超龄库存**：创建移除订单（Removal Order），设置自动移除规则
2. **提升售罄率**：加大广告投放、设置 Coupon、降价促销
3. **消灭滞销品**：Outlet Deal、捐赠、清算（Liquidation）
4. **保持在库率**：畅销品提前补货，不断货
5. **优化 SKU 数量**：关闭不盈利的 SKU，减少管理复杂度

### 3.5 仓储费规避策略

| 策略 | 操作 | 预期效果 |
|------|------|---------|
| **精准备货** | 控制 FBA 库存在 30-60 天周转 | 避免超龄仓储费 |
| **3PL 缓冲** | 大批量存 3PL，少量多频转 FBA | 降低 FBA 仓储费 50%+ |
| **旺季前清库存** | 9 月前清理慢销品（Q4 仓储费翻倍） | 避免 10-12 月高仓储费 |
| **移除计划** | 超 90 天未售的设置自动移除 | 避免超龄附加费 |
| **FBA 清算** | 利用 Amazon Outlet / Liquidation | 回收部分成本 |
| **AWD 中转** | 使用 Amazon Warehousing & Distribution | 大批量低成本仓储 |

### 3.6 滞销库存清理手册

当产品超过 90 天未售出，按以下优先级处理：

```
滞销库存处理优先级
│
├── 1. 降价促销（保住利润）
│   ├── 设置 Coupon（15-30% off）
│   ├── 创建 Lightning Deal / Best Deal
│   └── 调低售价 10-20%
│
├── 2. 广告冲量
│   ├── 开启 Sponsored Products 自动广告
│   └── 竞价调高 30-50%，短期冲量
│
├── 3. 站外清库存
│   ├── 社交媒体 Deal 群组（Slickdeals 等）
│   ├── 站外 Deal 网站引流
│   └── 独立站清仓专区
│
├── 4. Amazon 清算
│   ├── Outlet Deal（Amazon 折扣频道）
│   └── Liquidation Program（批量清算）
│
├── 5. 移除到海外仓
│   ├── 创建移除订单
│   └── 转到 3PL 仓，用于 FBM 发货或其他平台
│
└── 6. 弃置（最后手段）
    ├── Amazon 销毁（$0.15-$0.30/件）
    └── 捐赠（Amazon FBA Donations）
```
