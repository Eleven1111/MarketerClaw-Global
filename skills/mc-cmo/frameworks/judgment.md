# 判断框架（跨境版 v2）

mc-cmo 在 🟡 Light Path 和 🔴 Deep Path 中使用此框架做出判断。🟢 Fast Path 不加载此文件。

---

## 快速检查清单（Light Path 使用）

🟡 轻判断路径执行以下 3 项检查，任一项触发则升级为 🔴 深度介入：

### 1. 成本是否已知？
- 用户是否提供了产品成本、目标市场、物流方式？
- **已知** → 通过
- **未知 + 用户请求的是推广/投放/备货类技能** → 升级为 🔴
- **未知 + 用户请求的是选品/调研/竞品类技能** → 通过（这些技能本身就是在建立数据基础）
- **Brand Brain 有数据**（`products.md` 中已有成本信息）→ 通过

### 2. 风险等级？
- **低风险**（分析、监控、信息查询）→ 通过
- **中风险**（Listing 优化、文化洞察）→ 通过，但附带 CMO 建议
- **高风险**（广告投放、备货决策、新市场进入）→ 升级为 🔴

风险分类：
| 低风险 | 中风险 | 高风险 |
|--------|--------|--------|
| mc-compete | mc-listing | mc-ads（预算决策） |
| mc-monitor | mc-insight | mc-logistics（大批备货） |
| mc-selection（纯调研） | mc-sop | mc-cost（定价决策） |
| mc-compliance（查询） | mc-convert（审计） | 新市场进入 |
| mc-dashboard（查看） | mc-creative | mc-selection（决策阶段） |
| mc-diagnose（体检） | mc-social | mc-launch（预算分配） |
| | mc-retain（分析） | mc-retain（大规模投入） |

### 3. 合规检查？
- 目标市场的产品是否涉及特殊认证？（FDA、CE、FCC 等）
- **不涉及或已确认** → 通过
- **涉及且未确认** → CMO 提醒（不升级为 🔴，但在 cmo_context 中标注）

## 危机信号检测（Deep Path 额外检查）

在 Deep Path 中额外检测危机信号，任一命中则激活危机模式：

| 危机模式 | 关键词信号 | 激活后策略 |
|---------|----------|----------|
| `cash_crisis` | 亏钱/现金流/持续亏损/快没钱了/止血 | 止血优先，限制非紧急建议 |
| `listing_crisis` | 被下架/差评激增/被跟卖/账号预警/侵权 | 合规优先，mc-compliance + mc-brand 紧急介入 |
| `stock_crisis` | 断货/库存积压/仓储费爆/滞销 | mc-logistics 紧急响应，mc-ads 暂停/缩量 |

## 完整判断流程（Deep Path 使用）

🔴 深度介入按以下 4 步执行：

### Step A: 商业可行性
- 这个品/方向能赚钱吗？
- 成本结构是否清晰？（检查 `brand-brain/products.md`）
- 市场容量是否足够？
- 竞争格局是否允许新进入者？（检查 `brand-brain/competitors.md`）

### Step B: 前置条件检查
- 产品成本和利润模型是否已建立？
- 目标市场的消费者画像是否清晰？
- 竞品分析是否到位？
- 合规要求是否已确认？
- 供应链是否就绪？

缺失的前置条件按重要性排序，向用户建议补全路径。

### Step C: 风险评估
- 资金风险：需要投入多少？回本周期？
- 库存风险：备货量是否合理？滞销怎么处理？
- 合规风险：产品是否可能被下架？
- 竞争风险：大卖是否会跟进？价格战概率？

### Step D: 策略建议
基于 A/B/C 的分析，给出：
1. **推荐路径**：先算账、再选品、然后 Listing、最后推广
2. **理由**：跨境的核心逻辑是"先确定能赚钱，再考虑怎么卖"
3. **风险提示**：跳过成本计算直接推广 = 盲目烧钱
4. **预期产出**：每步会得到什么数据，如何支撑下一步决策

## 成本标签规范（全局生效）

所有技能的每条可执行建议必须附带成本标签：

```
💰 预算：$X    ⏱ 见效：X 周    👤 执行：卖家自己 / VA / 服务商
```

mc-cmo 在 cmo_context 中提醒目标技能遵守成本标签规范。
