## Module 6: 选品报告交付

### 6.1 报告结构

完整选品报告包含以下部分：

```
1. 执行摘要（Executive Summary）
2. 品类机会扫描（Module 1 产出）
3. 竞争格局分析（Module 2 产出）
4. 需求验证（Module 3 产出）
5. 选品决策矩阵（Module 4 产出）
6. 供应链初判（Module 5 产出）
7. 风险登记簿
8. 反共识检验
9. 行动计划
```

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  严重度: {高/中/低}

  影响: {具体影响描述}
  缓解方案: {具体可执行的缓解措施}
  监控指标: {如何监控这个风险}
```

**常见风险类型参考**：

| 风险类别 | 示例 |
|---------|------|
| 竞争风险 | 头部卖家降价挤压、大品牌入场、中国卖家涌入 |
| 供应链风险 | 原材料涨价、供应商断供、产品质量不稳定 |
| 合规风险 | 专利侵权投诉、平台政策变化、目的国法规变更 |
| 市场风险 | 需求萎缩、季节性误判、汇率波动 |
| 运营风险 | FBA 库存限制、差评冲击、账号关联风险 |

### 6.4 反共识检验（GO / CONDITIONAL 决策必做）

每条核心结论（尤其最终决策为 GO / CONDITIONAL 时）必须附**最强反驳证据**——
主动找推翻自己结论的数据，而不是只收集支持项：

```
| 核心结论 | 最强反驳证据 | 为什么结论仍然成立（或需修正） |
|---------|-------------|------------------------------|
```

- 找不到反驳证据时写"未找到——已搜索 {关键词}"，不允许留空。
- 若反驳证据足够强，直接修正核心结论（GO 降级为 CONDITIONAL / PASS），不要硬撑原判断。
- PASS 决策可免做（放弃判断不需要对抗自证偏误）。

### 6.5 行动计划

如果决策为 GO 或 CONDITIONAL，提供明确的下一步行动：

```
行动计划（GO 决策后 30 天路线图）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

第 1 周: 供应商确认
  · 联系 3-5 家 1688/Alibaba 供应商
  · 索要样品（至少 2 家以上）
  · 确认 MOQ、单价、交期
  · 目标产出: 锁定 1-2 家候选供应商

第 2 周: 产品确认 + 认证
  · 收到样品，品质评估
  · 确认产品规格和包装方案
  · 启动所需认证（如有）
  · 目标产出: 确认最终产品方案

第 3 周: 品牌 + Listing 准备
  · 注册/确认品牌（Brand Registry）
  · 用 mc-listing 技能准备 Listing 文案
  · 产品摄影方案确定
  · 目标产出: Listing 文案和图片方案就绪

第 4 周: 下单 + 物流安排
  · 向供应商下首批订单
  · 联系货代安排头程物流
  · 创建 FBA Shipment
  · 用 mc-ads 技能准备广告策略
  · 目标产出: 首批订单生产中，物流方案确认

后续关键里程碑:
  · 第 6-8 周: 收货验货，发货到 FBA
  · 第 8-10 周: 上架开卖，启动广告
  · 第 10-14 周: 优化广告，申请 Vine Review
  · 第 14-18 周: 评估首批销售数据，决定是否补货
```
