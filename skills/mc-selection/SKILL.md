## 2026 认知校准（每次选品前必读）

> 来源：r/FulfillmentByAmazon 高赞实战帖 + 8年私标老手复盘。违背这三条认知，评分再高的产品也会失败。

**幻觉 1：选品是关键**
Amazon 不是"找到一个爆款"的游戏，是"比下一个卖家运营得更好"的游戏。流量从来不是问题——Amazon 已经有流量了。真正的问题是：你的 Listing 能不能把流量转化成利润？

**幻觉 2：竞争是最大威胁**
现金流才是。断货、压库存、算错利润率——这三件事比任何算法变动都更快杀死一个店铺。选品阶段必须算清现金流，不只是利润率。

**幻觉 3：市场饱和了**
饱和的不是市场，是同质化。一模一样的产品、图片、文案，价格成了唯一武器，那是一场你赢不了的战争。几乎所有品类都有差异化空间，关键是找到它。

**2026 资金现实**（来自 8 年老手）：
- 认真做 1 个 SKU 的最低门槛：**$10,000–$15,000**（含库存、摄影、广告预热）
- 有容错空间的合理起步：**$50,000+**（含 1–2 个失败品的缓冲）
- Amazon 平均抽走卖家**收入的 50% 以上**（FBA 费 + 广告 + 平台佣金合计）
- 单 SKU 靠规模替代全职收入的时代基本结束；品牌化 + 多 SKU 才是稳健路径

**何时跳过本技能直接放弃**：
- 目标售价 < $15（利润空间不足以支撑 FBA 费 + 广告）
- 品类 Top 3 评论数合计 > 15,000 且产品无法显著差异化
- 毛利率测算 < 28%（净利润 = 0 或亏损）

## Module 1: 品类机会扫描

### 1.1 市场规模估算（TAM/SAM/SOM）

跨境电商选品的市场规模不等于传统意义上的"全球市场"。按以下三层递进估算：

| 层级 | 定义 | 跨境电商含义 | 估算方法 |
|------|------|------------|---------|
| TAM | 总可寻址市场 | 该品类在目标平台所在国家的线上零售总额 | 行业报告 + 平台品类销售额 |
| SAM | 可服务市场 | 该品类在目标平台（如 Amazon US）上的总销售额 | BSR Top 100 月销 × 均价 × 放大系数 |
| SOM | 可获得市场 | 新进入者在 12 个月内可合理获取的市场份额 | SAM × 新卖家合理市占率（通常 0.5%-3%） |

**SAM 快速估算公式**（Amazon）：

```
SAM = BSR Top 100 月均销量之和 × 品类均价 × 1.5（长尾放大系数）
```

**月均销量估算方法**：

| 方法 | 说明 | 精度 |
|------|------|------|
| Jungle Scout 数据 | 直接读取工具估算 | ★★★★ |
| BSR 公式法 | 根据 BSR 排名与品类系数换算 | ★★★ |
| Review 增速法 | 月新增 Review 数 ÷ Review 率（约 1%-3%） | ★★ |
| 库存追踪法 | 连续 7 天追踪库存数量变化 | ★★★★★ |

**SOM 新卖家合理市占率参考**：

| 卖家类型 | 无品牌优势 | 有品牌/差异化 | 有站外资源 |
|---------|-----------|-------------|-----------|
| 新手 | 0.3%-0.5% | 0.5%-1% | 1%-2% |
| 中阶 | 0.5%-1% | 1%-2% | 2%-3% |
| 资深 | 1%-2% | 2%-5% | 3%-8% |

### 1.2 需求信号检测

从多维度验证品类需求是否"真实且持续"：

**搜索量趋势分析**：

| 信号 | 正面（机会） | 负面（风险） |
|------|------------|------------|
| 搜索量趋势 | 稳定或上升（12 个月环比增长 >10%） | 持续下降（12 个月环比下降 >15%） |
| 搜索量规模 | 核心词月搜索量 >10,000 | 核心词月搜索量 <2,000 |
| 长尾词丰富度 | 相关长尾词 >50 个 | 只有 1-2 个核心词，无长尾 |
| 新词出现率 | 近 6 个月出现新搜索词 | 搜索词固化，无新词出现 |

**BSR 分布分析**：

```
BSR 分布健康度 = BSR Top 10 中非品牌卖家占比

> 50% → 市场开放，新卖家有机会
30%-50% → 中等竞争，需要差异化
< 30% → 品牌垄断，慎入
```

**需求信号评分卡**：

| 指标 | 5 分（极好） | 4 分 | 3 分 | 2 分 | 1 分（极差） |
|------|------------|------|------|------|------------|
| 核心词搜索量 | >50,000/月 | 20,000-50,000 | 10,000-20,000 | 5,000-10,000 | <5,000 |
| 搜索量趋势 | 年增长 >30% | 增长 10%-30% | 持平 ±10% | 下降 10%-30% | 下降 >30% |
| 长尾词数量 | >100 个 | 50-100 | 20-50 | 10-20 | <10 |
| 新词出现率 | 每月出现新词 | 季度出现新词 | 半年出现新词 | 一年无新词 | 搜索词萎缩 |

### 1.3 供需缺口分析

核心问题：搜索的人多不多？卖的人多不多？比值是否健康？

**供需比公式**：

```
供需比 = 核心关键词月搜索量 ÷ 品类在售商品数量（搜索结果数）

> 8.0 → 严重供不应求（蓝海信号强）
3.0 - 8.0 → 供需平衡偏供不应求（机会窗口）
1.0 - 3.0 → 供需基本平衡（需差异化）
< 1.0 → 供大于求（红海警告）
```

**补充验证指标**：

| 指标 | 蓝海信号 | 红海信号 |
|------|---------|---------|
| 品类在售商品数 | <2,000 个结果 | >50,000 个结果 |
| 新品上架速度 | 月均 <50 个新品 | 月均 >500 个新品 |
| 头部集中度 | Top 10 销量占比 <40% | Top 10 销量占比 >70% |
| 广告密度 | 搜索首页赞助位 <30% | 搜索首页赞助位 >50% |

### 1.6 季节性评估

**季节性分级**：

| 等级 | 定义 | 判断标准 | 选品建议 |
|------|------|---------|---------|
| S1 - 无季节性 | 全年需求稳定 | 月搜索量波动 <20% | 首选品类，现金流稳定 |
| S2 - 轻季节性 | 有旺淡季但淡季仍可卖 | 月搜索量波动 20%-50% | 可做，备货节奏要控制 |
| S3 - 强季节性 | 明显旺季窗口 | 月搜索量波动 50%-80% | 有经验者可做，新手慎选 |
| S4 - 极端季节性 | 仅特定时段有需求 | 月搜索量波动 >80% | 不建议作为主力产品 |

**关键日历节点（Amazon US）**：

| 月份 | 重要节点 | 关联品类示例 |
|------|---------|-------------|
| 1-2 月 | 新年决心、情人节 | 健身器材、收纳用品、情人节礼品 |
| 3-4 月 | 春季大扫除、复活节 | 清洁用品、园艺工具、户外装饰 |
| 5 月 | 母亲节 | 礼品、珠宝、厨房小家电 |
| 6-7 月 | Prime Day、暑假 | 电子产品、户外用品、旅行配件 |
| 8-9 月 | 返校季 | 文具、背包、电子设备 |
| 10 月 | 万圣节 | 装饰品、服装配件、糖果 |
| 11 月 | 黑五 + 网一 | 全品类爆发，尤其电子和家居 |
| 12 月 | 圣诞节 | 礼品全品类、装饰品、玩具 |

**其他市场日历差异提醒**：
- **JP**：1 月新年福袋、3 月毕业季、7 月 Prime Day、12 月圣诞
- **DE/EU**：11 月黑五、12 月圣诞集市、复活节比 US 更重要
- **Shopee SEA**：9.9/10.10/11.11/12.12 大促节奏、斋月（穆斯林市场）
- **TikTok Shop**：跟随平台活动节奏，短视频内容驱动，季节性受内容热度影响更大

## Module 2: 竞争格局分析

### 2.1 品类集中度分析

**核心指标：CR10（Top 10 卖家市场份额）**

```
CR10 = Top 10 卖家月销售额之和 ÷ 品类总月销售额 × 100%

> 70% → 高度集中，寡头市场（进入难度极高）
40%-70% → 中等集中（有机会但需差异化）
< 40% → 分散市场（新卖家友好）
```

**品牌 vs 白牌比例**：

| 指标 | 新卖家友好 | 中性 | 不友好 |
|------|-----------|------|-------|
| Top 20 中品牌占比 | <30% | 30%-60% | >60% |
| 品牌注册卖家占比 | <40% | 40%-70% | >70% |
| 头部是否有亚马逊自营 | 否 | 有但不在 Top 5 | Top 3 有亚马逊自营 |

**卖家类型分布表**：

| 卖家类型 | 数量 | 占比 | 平均月销量 | 平均 Review 数 |
|---------|------|------|----------|--------------|
| 国际大品牌 | | | | |
| 亚马逊自营 | | | | |
| 品牌注册卖家 | | | | |
| 白牌/无品牌卖家 | | | | |
| 中国卖家 | | | | |

### 2.2 进入壁垒评估

逐项评估以下进入壁垒，每项 1-5 分（1=无壁垒，5=极高壁垒）：

**资金壁垒**：

| 分数 | 首批备货成本（含 FBA 头程） | 说明 |
|------|---------------------------|------|
| 1 | <$2,000 | 轻资产入场 |
| 2 | $2,000-$5,000 | 小额投入 |
| 3 | $5,000-$15,000 | 中等投入 |
| 4 | $15,000-$50,000 | 较大资金量 |
| 5 | >$50,000 | 重资产品类 |

**认证壁垒**：

| 分数 | 所需认证 | 说明 |
|------|---------|------|
| 1 | 无需认证 | 普通日用品 |
| 2 | 简单合规（CPC 等） | 儿童产品基础认证 |
| 3 | 标准认证（UL/FCC 等） | 电子产品常规认证 |
| 4 | 严格认证（FDA/EPA 等） | 食品接触、杀虫类 |
| 5 | 极高门槛（医疗器械、处方类） | 基本不建议新手 |

**知识产权壁垒**：

| 分数 | IP 风险程度 | 说明 |
|------|-----------|------|
| 1 | 品类通用，无 IP 风险 | 基础日用品 |
| 2 | 存在少量外观专利 | 需避开特定设计 |
| 3 | 品类有活跃的 IP 诉讼 | 需专利排查 |
| 4 | 核心功能受专利保护 | 创新空间受限 |
| 5 | 品类被少数专利覆盖 | 极高侵权风险 |

**MOQ 壁垒**：

| 分数 | 起订量 | 说明 |
|------|-------|------|
| 1 | <100 件 | 极低门槛 |
| 2 | 100-500 件 | 小批量测试可行 |
| 3 | 500-2,000 件 | 中等起订量 |
| 4 | 2,000-5,000 件 | 需较大资金 |
| 5 | >5,000 件 | 需工厂关系或大资金 |

**进入壁垒综合评分**：

```
进入壁垒分 = (资金壁垒 × 0.3) + (认证壁垒 × 0.25) + (IP 壁垒 × 0.25) + (MOQ 壁垒 × 0.2)

1.0-2.0 → 低壁垒（新手友好）
2.0-3.0 → 中等壁垒（需准备）
3.0-4.0 → 高壁垒（中阶以上卖家）
4.0-5.0 → 极高壁垒（资深卖家 + 资源匹配）
```

### 2.3 Review 壁垒分析

Review 数量是新卖家最直观的竞争门槛。

**Review 壁垒评估表**：

| 指标 | 低门槛 | 中门槛 | 高门槛 | 极高门槛 |
|------|-------|-------|-------|---------|
| Top 10 平均 Review 数 | <200 | 200-1,000 | 1,000-5,000 | >5,000 |
| 首页最低 Review 数 | <50 | 50-200 | 200-500 | >500 |
| Top 10 平均评分 | <4.0 | 4.0-4.3 | 4.3-4.5 | >4.5 |
| 新品上首页所需 Review | <30 | 30-100 | 100-300 | >300 |

**新卖家 Review 追赶速度估算**：

```
合规 Review 获取速度:
  Vine 计划: 约 30 个/批次（需 Brand Registry）
  自然 Review 率: 订单量的 1%-3%
  Request a Review 按钮: 可提升 Review 率至 3%-5%

追赶时间 = (首页最低 Review 数 - 30) ÷ (月销量 × Review 率)
```

## Module 3.5: 差异化定义（选品不通过此关不进入决策矩阵）

> 来源：Reddit 实战帖核心结论。2026 年的 Amazon，同质化产品必死。
> **做完需求验证后，必须能回答"我的差异化是什么"，才能进入决策矩阵。**

### 三条有效差异化路径

**路径 1：捆绑套件（Bundle）**

把单品打包成解决方案，形成唯一 Listing，无法被直接比价。

| 指标 | 效果 |
|------|------|
| 客单价提升 | +20%-30% |
| 买家 LTV | 高 2.7 倍（vs 单品买家） |
| 竞争压力 | 大幅降低（无直接同款可比价） |

执行要点：
- 找 2-3 个强关联的互补产品打包（同一使用场景）
- 套装定价 = 各单品市场价之和 × 0.8-0.9（让用户感知"占便宜"）
- 3PL 或工厂提前做好套装包装，不在 FBA 仓里人工组合（成本高）

**路径 2：痛点升级（Pain Point Fix）**

从竞品差评中找到高频痛点，把解决方案做进产品，变成差评对比的武器。

执行步骤：
1. 读品类 Top 5-10 卖家的 1-3 星差评，统计高频词
2. 找到"提及率 >15% 且我方可以解决"的痛点
3. 产品改进方向锁定该痛点（材质/结构/尺寸/配件）
4. Listing 主 Bullet Point #1 直接点名解决该痛点

示例：瑜伽垫差评"太滑" → 做加厚防滑版 → Bullet #1 写"Non-slip surface tested with 3x more grip than standard mats"

**路径 3：品类转移（Category Transfer）**

把 TikTok / Etsy / 线下热门产品带上 Amazon，利用平台信息差。

先发优势窗口期通常 3-6 个月，之后竞争对手涌入。

信号识别：
- TikTok 有大量相关内容但 Amazon 搜索结果 < 1,000 个
- Etsy 热销但 Amazon 同类词搜索量刚开始上升
- Google Trends 近 6 个月快速上升但 Amazon BSR 头部玩家评论数 < 200

### 差异化可行性评估

在进入 Module 4 前，完成以下 3 个问题的回答：

```
差异化检查
─────────────────────────────
1. 我打算用哪条路径差异化？
   [ ] 捆绑套件  [ ] 痛点升级  [ ] 品类转移  [ ] 其他：___

2. 具体差异化描述（一句话，用户视角）：
   "和竞品比，我的产品___，解决了___的问题"

3. 竞品能在 90 天内复制我的差异化吗？
   [ ] 不能（有独特性）  [ ] 能，但我有先发优势  [ ] 能，需要重新定义差异化

如果第 3 项选"能且无先发优势"→ 差异化不成立，返回 Module 3 重做。
─────────────────────────────
```

## Module 4: 选品决策矩阵

### 4.1 四维评分框架

**总分 = 需求分(30%) × 竞争分(25%) × 利润分(25%) × 可行性分(20%)**

每个维度 1-5 分，每分有明确判断标准。

### 4.2 需求维度评分（权重 30%）

| 分数 | 标准 | 典型情形 |
|------|------|---------|
| 5 | 需求强劲且增长 | 核心词搜索量 >30,000/月，年增长 >20%，长尾词丰富，跨平台验证一致 |
| 4 | 需求明确且稳定 | 核心词搜索量 10,000-30,000/月，趋势平稳或微涨，长尾词较多 |
| 3 | 需求存在但一般 | 核心词搜索量 5,000-10,000/月，趋势平稳，长尾词有限 |
| 2 | 需求较弱或下降 | 核心词搜索量 2,000-5,000/月，或搜索量在下降 |
| 1 | 需求不足或衰退 | 核心词搜索量 <2,000/月，或年下降 >20%，或需求无法验证 |

### 4.3 竞争维度评分（权重 25%，注意：分数越高=竞争越小=越有利）

| 分数 | 标准 | 典型情形 |
|------|------|---------|
| 5 | 竞争极低（蓝海） | CR10 <30%，Top 10 平均 Review <200，无强势品牌，供需比 >8 |
| 4 | 竞争较低 | CR10 30%-45%，Top 10 平均 Review 200-500，1-2 个品牌，供需比 5-8 |
| 3 | 竞争中等 | CR10 45%-60%，Top 10 平均 Review 500-1,500，有知名品牌但不垄断 |
| 2 | 竞争较高 | CR10 60%-75%，Top 10 平均 Review 1,500-5,000，品牌主导 |
| 1 | 竞争极高（红海） | CR10 >75%，Top 10 平均 Review >5,000，品牌垄断，广告费极高 |

### 4.4 利润维度评分（权重 25%）

| 分数 | 标准 | 典型情形 |
|------|------|---------|
| 5 | 高利润率 | 毛利率 >45%，售价 $25-$75，FBA 费用占比 <25% |
| 4 | 良好利润率 | 毛利率 35%-45%，定价灵活，广告空间充足 |
| 3 | 中等利润率 | 毛利率 25%-35%，需要控制广告花费 |
| 2 | 薄利 | 毛利率 15%-25%，价格战风险高，广告空间有限 |
| 1 | 微利或亏损 | 毛利率 <15%，或只能靠走量，或 FBA 费用吞噬利润 |

**利润率计算公式**：

```
毛利率 = (售价 - 产品成本 - 头程物流 - FBA 费 - 平台佣金 - 目标广告费) ÷ 售价 × 100%

各项参考值:
  产品成本: 售价的 20%-35%
  头程物流: $1-5/件（视体积重量）
  FBA 配送费: 按 Amazon 费率表（标准件 $3.22-$6.50+）
  平台佣金: 15%（大部分品类）
  目标广告费: 售价的 10%-20%（按 ACoS 25%-40% 估算）
```

### 4.5 可行性维度评分（权重 20%）

| 分数 | 标准 | 典型情形 |
|------|------|---------|
| 5 | 极高可行性 | 供应商多、MOQ 低(<200)、无需认证、体积小重量轻、非危险品 |
| 4 | 高可行性 | 供应商较多、MOQ 合理(<500)、基础认证即可、标准物流 |
| 3 | 中等可行性 | 供应商有限、MOQ 中等(500-2000)、需要特定认证、物流无特殊困难 |
| 2 | 较低可行性 | 供应商少、MOQ 大(>2000)、认证复杂、超大件或带电池 |
| 1 | 极低可行性 | 定制开模、MOQ 极大、多重严格认证、危险品、冷链等特殊物流 |

### 4.6 红线检查（一票否决）

以下任一项触发即标记为 **PASS（放弃）**，无论评分多高：

| 红线 | 说明 | 检查方法 |
|------|------|---------|
| 专利侵权高风险 | 核心功能或外观受专利保护 | 搜索 USPTO/WIPO 专利库 |
| 平台禁售/限制品类 | Amazon 禁止或需要特殊审批 | 查看 Amazon Restricted Products 列表 |
| 强制召回历史 | 品类有 CPSC 召回记录 | 搜索 CPSC.gov 召回数据库 |
| 法规壁垒不可逾越 | 如 FDA 510(k) 审批 | 评估认证时间和成本 |
| 物流不可行 | 如需冷链、危险品运输成本过高 | 咨询货代 |
| 售后噩梦品类 | 退货率 >15%、高频售后纠纷 | 查看品类平均退货率 |
| 头部垄断且无差异化空间 | 市场被 1-2 个品牌垄断且产品同质化 | Module 2 分析结果 |

### 5.2 成本结构基准

**产品成本构成**：

```
产品出厂价 (FOB)
  = 原材料成本
  + 生产加工费
  + 包装费（内包装 + 外箱）
  + 品质检验费（验货费分摊）
  + 工厂利润（通常 10%-20%）

到手成本（Landed Cost）
  = FOB 价
  + 国内运输到港/仓
  + 报关/商检费
  + 海运/空运费
  + 目的国关税
  + 目的国清关费
  + 尾程派送到 FBA 仓
```

**成本占比基准（Amazon FBA 标准件）**：

| 成本项 | 占售价比例 | 备注 |
|-------|----------|------|
| 产品成本（FOB） | 15%-30% | 目标 <25% |
| 头程物流（到 FBA） | 5%-15% | 海运约 5%-8%，空运约 10%-15% |
| FBA 配送费 | 10%-20% | 按体积重量计算 |
| 平台佣金 | 15% | Amazon 大部分品类 |
| 广告费 | 8%-20% | 新品期偏高 |
| 其他费用 | 3%-5% | 仓储、退货、损耗等 |
| **目标净利润** | **15%-30%** | **低于 15% 不建议做** |

### 5.4 物流复杂度评估

| 因素 | 低复杂度（1 分） | 中复杂度（3 分） | 高复杂度（5 分） |
|------|-----------------|-----------------|-----------------|
| 尺寸 | 标准件（<18"任一边） | 大标准件 | 超大件/超重 |
| 重量 | <2 lbs | 2-20 lbs | >20 lbs |
| 易碎性 | 不易碎 | 一般易碎 | 极易碎（玻璃/陶瓷） |
| 带电池 | 无电池 | 内置锂电池 | 纯电池/超大电池 |
| 液体/粉末 | 无 | 含少量液体 | 大量液体/气雾剂 |
| 温度敏感 | 不敏感 | 需避免极端温度 | 需冷链 |
| 危险品分类 | 非危险品 | 有限数量危险品 | 完全危险品 |

**物流复杂度评分**：

```
物流复杂度分 = 各因素最高分

1-2 分 → 简单物流，FBA 标准流程
3 分 → 中等复杂，需要注意包装和申报
4 分 → 较复杂，需要专业货代，可能有额外费用
5 分 → 高复杂，需评估是否值得做（物流成本可能吞噬利润）
```

| 定制化 | 标准品 | 需要定制/个性化 | 需要贴标/改包装 |

## 适用场景

- **新手卖家**：第一次选品，不知道从何入手，需要系统化的选品框架
- **品类切换**：现有品类遇到瓶颈，寻找新品类机会
- **多品评估**：手上有 2-5 个产品方向，需要量化对比做出决策
- **机会验证**：看到某个"热门品类"想入场，需要冷静评估风险
- **跨平台扩展**：从 Amazon 扩展到 Shopee/TikTok Shop，需要评估新平台机会
- **供应链评估**：已确定品类，需要评估供应链可行性和资金需求

## 执行步骤

### Step 1: 明确选品方向

收集用户的选品需求：
- 用户已有具体产品方向？还是需要从品类大方向开始？
- 明确目标平台和站点（默认 Amazon US）
- 了解用户背景（新手/中阶/资深，预算范围，已有资源）
- 确认评估范围（单品 vs 多品对比）

### Step 2: 品类机会扫描（Module 1）

执行市场规模估算、需求信号检测、供需缺口分析、季节性评估。输出品类机会评分卡。如果机会评分极低（<2/5），提前告知用户风险，由用户决定是否继续深入。

### Step 3: 竞争格局分析（Module 2）

执行品类集中度分析、进入壁垒评估、Review 壁垒分析、价格带分析。输出竞争格局矩阵。如果发现红线问题（如专利垄断），立即标记。

### Step 4: 需求验证（Module 3）

执行关键词簇分析、Review 挖掘、用户之声提取、跨平台交叉验证。输出需求验证报告。重点关注差异化机会和未满足需求。

### Step 5: 决策与供应链（Module 4 + Module 5）

基于前三步数据填写决策矩阵，执行红线检查。如果决策为 GO 或 CONDITIONAL，继续执行供应链初判。如果 PASS，直接跳到 Step 6 输出结论。

### Step 6: 报告交付（Module 6）

整合所有模块产出，输出完整选品报告，包含执行摘要、风险登记簿、行动计划。

### 示例：评估"宠物自动喂食器"在 Amazon US 的选品机会

**用户输入**：
> 想做宠物自动喂食器，Amazon US 市场，预算 $10,000 左右，新手卖家，帮我评估值不值得做。

**执行流程**：

**Step 1 — 明确方向**：
- 产品：宠物自动喂食器（automatic pet feeder）
- 平台：Amazon US
- 预算：~$10,000
- 卖家类型：新手

**Step 2 — 品类机会扫描**（输出品类机会评分卡）：

```

  TAM: $2.1B（美国宠物用品在线零售）
  SAM: ~$180M/年（Amazon US 自动喂食器品类）
  SOM: ~$0.5M-$1M/年（新卖家 12 个月合理预期）

需求信号:
  核心词搜索量    ████████░░ 45,000/月 — 4/5
  搜索量趋势      ███████░░░ 年增长约 12% — 4/5
  长尾词丰富度    ████████░░ 80+ 相关词 — 4/5
  新词出现率      ██████░░░░ 季度出现新词（智能/WiFi 相关） — 3/5
  需求信号综合分: 3.8/5

供需缺口:
  供需比: 3.2（供需基本平衡，偏供不应求）
  品类在售商品数: ~14,000

  供需缺口评价: 有细分机会但非大蓝海

季节性:
  等级: S2 - 轻季节性
  旺季窗口: 11-12 月（节日送礼）、6-7 月（暑期出行）
  备货提醒: 9 月前备足 Q4 库存

品类机会综合评分: 3.5/5
结论: 需求确定且增长，市场体量大，但非蓝海，需差异化切入
```

**Step 3 — 竞争格局**（关键数据）：
- CR10: ~55%（中等集中）
- Top 10 平均 Review: ~8,500 条（Review 壁垒高）

- 品牌占比: Top 20 中约 60% 为品牌注册卖家

- 进入壁垒综合分: 3.2/5（中等偏高，电子产品需 FCC 认证，带电池）

- Review 高频差评: WiFi 连接不稳定（38%）、食物卡住（25%）、噪音大（18%）
- 未满足需求: 双猫分食控制、小体积适合小户型、静音设计
- 差异化机会: "静音 + 双宠分食"方向在 Review 中被频繁提及但市场上方案少

**Step 5 — 决策矩阵**：

## 常见陷阱

### 陷阱 1: 唯搜索量论
只看核心关键词搜索量大就认为"好卖"，忽略竞争强度和利润空间。搜索量 100,000/月但 Top 10 全是大品牌、毛利率 15%，不如搜索量 8,000/月但竞争低、毛利率 40% 的品类。**搜索量是筛选起点而非决策终点。**

### 陷阱 2: 忽略 Review 壁垒的时间成本
看到品类"利润高"就冲进去，没计算追上 Top 10 的 5,000 条 Review 需要多少时间和成本。新品期 Review 增长缓慢，转化率低，广告费吞噬利润，现金流断裂是新手最常见的失败原因。**Review 追赶时间 >12 个月的品类，新手应谨慎。**

### 陷阱 3: 用 Google Trends 替代 Amazon 搜索量
Google Trends 反映的是信息搜索需求，不等于购买需求。某个话题在 Google 上很火，但用户可能在线下购买或根本不在 Amazon 搜索。**Google Trends 只能做辅助验证，不能作为需求量化的唯一依据。**

### 陷阱 4: 低估物流和合规成本
选品时只算产品成本和 FBA 费用，忽略头程物流涨价、关税、认证费用、首批样品费等隐性成本。尤其是带电池的产品，危险品申报和特殊物流渠道可能增加 30%-50% 的物流成本。**用 Landed Cost（到手成本）而非 FOB 价格算利润。**

### 陷阱 5: 幸存者偏差——只看成功案例
在卖家论坛或培训课上看到"某品类月销 $100,000"就跟风入场，没有看到同品类几百个失败卖家。BSR 排名靠前的是幸存者，搜索结果第 5 页之后的才是大多数新卖家的命运。**看品类中位数表现，而非头部表现。**

### 陷阱 6: 忽视季节性备货节奏
季节性产品如果备货时间算错，旺季到了货还在海上，等货到了旺季已过。或者旺季过度备货，淡季仓储费吞噬利润。**季节性品类必须倒推备货时间线：旺季前 2-3 个月到货入仓。**

### 陷阱 7: 一个数据源做决策
只用 Jungle Scout 或只用 Helium 10 的数据就下结论。不同工具的估算偏差可能达 30%-50%。**至少用 2 个数据源交叉验证，对关键数据保持怀疑态度。**

## 验证清单

完成选品研究后，逐项确认：

- [ ] 市场规模已量化（TAM/SAM/SOM 均有估算，数据来源注明）
- [ ] 需求信号经过多维度验证（搜索量 + 趋势 + 长尾词 + 跨平台）
- [ ] 竞争格局有具体数据支撑（CR10、Review 门槛、品牌占比、价格带）
- [ ] 进入壁垒已逐项评估（资金、认证、IP、MOQ）
- [ ] Review 追赶时间已估算，新手可接受（<12 个月）
- [ ] 利润模型已计算，含所有成本项（毛利率 >25%）
- [ ] 红线检查已完成，无一票否决项
- [ ] 供应链已初步验证（有供应商、MOQ 合理、物流可行）
- [ ] 季节性已评估，备货时间线合理
- [ ] 差异化方向已明确（不是简单跟卖）
- [ ] 首批投入资金已测算，用户预算可覆盖
- [ ] 风险 Top 3 已识别，缓解方案已制定
- [ ] 数据来源至少 2 个以上交叉验证
- [ ] 最终决策有明确理由（GO/CONDITIONAL/PASS）

## 核心问题

选品研究过程中，持续追问以下问题：

**关于需求**：
- 这个需求是真实的还是被营销创造的？
- 需求在增长还是衰退？增长的驱动力是什么？
- 用户为什么要在网上买这个东西（而不是线下）？
- 长尾需求中是否隐藏着被忽视的细分机会？

**关于竞争**：
- 如果我是品类第一名，我最害怕什么样的新进入者？
- 新卖家在这个品类凭什么能拿到订单？
- 头部卖家的护城河到底是品牌、Review、价格还是产品力？
- 这个品类未来 12 个月竞争会变好还是变差？

**关于利润**：
- 如果头部卖家发起价格战，我的利润还能撑住吗？
- 广告费会随竞争加剧持续上升吗？
- 有没有被忽视的隐性成本（退货、仓储、质保）？
- 人民币对目标货币汇率波动 10% 对利润的影响？

**关于可行性**：
- 以我当前的资源（资金、经验、团队），这个产品我做得了吗？
- 从下单到赚回本金需要多久？现金流撑得住吗？
- 如果第一批 500 件卖不动，最坏的情况是什么？
- 有没有更轻资产的方式先验证这个市场（如 FBM 测品）？

**关于差异化**：
- 我的产品凭什么让用户放弃已有 3,000 条 Review 的竞品来买我？
- 这个差异化是用户真正在乎的，还是我自己觉得重要？
- 差异化能体现在 Listing 上吗？用户在购买前能感知到吗？
- 竞品多快能复制我的差异化？

## 交互规则

1. **先收集，再输出** — 没有足够品类信息时追问，不凭空编造市场数据
2. **数据透明** — 每个数据点标明来源和置信度，估算值明确标注"估算"
3. **分步交付** — 全链路选品时分模块输出，每个模块输出后暂停确认再继续
4. **保守为先** — 宁可低估机会，不可高估。新手卖家的钱输不起
5. **不替代工具数据** — 搜索量、BSR、月销量等硬数据需要用户从专业工具获取
6. **多方案对比** — 有多个产品方向时，主动用对比矩阵帮用户决策
7. **合规提醒** — 发现潜在合规/IP 风险时主动提醒，建议用 mc-compliance 深度审查
