[2] ...

## Module 1: Amazon PPC 策略

### Campaign 架构

按三层结构搭建，每个 SKU 一套完整体系：

```
├── Auto Campaign（自动广告）
│   ├── Close Match     — 紧密匹配
│   ├── Loose Match     — 宽泛匹配
│   ├── Substitutes     — 同类商品
│   └── Complements     — 互补商品
│
├── Manual Campaign — Broad（手动广泛）
│   └── 核心词 + 长尾词，匹配类型 Broad
│
├── Manual Campaign — Phrase（手动词组）
│   └── 从 Auto 和 Broad 中跑出的高转化词
│
├── Manual Campaign — Exact（手动精准）
│   └── 已验证的高转化精准词，最高竞价
│
└── Sponsored Brands / Video（品牌广告）
    └── 品牌词防御 + 品类词截流
```

### 关键词策略

关键词来源优先级：

1. **mc-listing 输出的关键词**：如已执行 mc-listing，直接复用其关键词矩阵
2. **竞品 ASIN 反查**：通过竞品 Listing 提取高频词
3. **搜索词报告挖掘**：Auto Campaign 跑 2 周后提取
4. **品类相关长尾词**：搜索量 1000-5000 的长尾词，竞争度低，优先投放

关键词分层：

| 层级 | 关键词类型 | 匹配方式 | 竞价策略 |
|------|-----------|---------|---------|
| Tier 1 | 品牌词 | Exact | 高竞价，防御为主 |
| Tier 2 | 核心品类词（搜索量 >10K） | Exact + Phrase | 中高竞价，抢排名 |
| Tier 3 | 长尾词（搜索量 1K-10K） | Broad + Phrase | 中等竞价，跑量 |
| Tier 4 | 场景词 / 属性词 | Broad | 低竞价，测试 |

### 竞价管理框架

基础竞价设定：

```
建议出价 = 品类平均 CPC × 调整系数

调整系数：
  新品期（0-3月）  → 1.2-1.5x（需要曝光，可接受亏损）
  成长期（3-6月）  → 1.0-1.2x（平衡曝光与效率）
  成熟期（6-12月） → 0.8-1.0x（效率优先）
  收割期（12月+）  → 0.6-0.8x（利润最大化）
```

竞价调整规则（每周执行）：

| 条件 | 操作 |
|------|------|
| ACOS < 目标值 且 曝光充足 | 维持当前竞价 |
| ACOS < 目标值 且 曝光不足 | 提高竞价 10-15% |
| ACOS > 目标值 1.5 倍 | 降低竞价 15-20% |
| 点击 > 20 且 0 转化 | 暂停该关键词，加入否定 |
| CTR < 0.3% | 检查 Listing 主图和标题 |

### ACOS / TACOS 目标设定

按产品生命周期设定不同目标：

| 阶段 | ACOS 目标 | TACOS 目标 | 说明 |
|------|----------|-----------|------|
| Launch（0-3月） | 50-80% | 25-40% | 允许亏损换排名和评论 |
| Growth（3-6月） | 30-50% | 15-25% | 有机销量开始增长 |
| Mature（6-12月） | 15-30% | 8-15% | 广告占比下降，利润提升 |
| Harvest（12月+） | 10-20% | 5-10% | 品牌词为主，利润最大化 |

> TACOS = 广告花费 / 总销售额。TACOS 比 ACOS 更能反映广告对整体业务的贡献。

### 预算分配（Amazon 内部）

| Campaign 类型 | 新品期 | 成长期 | 成熟期 |
|--------------|--------|--------|--------|
| Auto | 30% | 15% | 10% |
| Manual Broad | 25% | 20% | 10% |
| Manual Phrase | 15% | 25% | 20% |
| Manual Exact | 20% | 30% | 45% |
| SB / SBV | 10% | 10% | 15% |

### 否定关键词策略

否定词来源：

1. **搜索词报告**：点击 > 15 且 0 转化的词 → 精准否定
2. **不相关品类词**：与产品无关的品类 → 词组否定
3. **竞品品牌词**（非防御目的）：转化率极低时 → 精准否定
4. **低价值修饰词**：`cheap`、`free`、`used` → 账户级否定

否定词审查频率：每周一次，新品期可每 3 天一次。

## Module 2: Amazon DSP & Sponsored Display

### DSP vs SD 决策矩阵

| 维度 | Sponsored Display (SD) | Amazon DSP |
|------|----------------------|------------|
| 最低预算 | 无门槛 | $10,000/月起（自助）或 $35,000/月（托管） |
| 定向能力 | 基础（ASIN、品类、浏览再营销） | 高级（人口统计、购买行为、生活方式） |
| 投放位置 | Amazon 站内为主 | Amazon 站内 + 站外（Twitch、IMDB 等） |
| 适用场景 | 所有卖家，基础再营销 | 品牌卖家，大预算，品牌建设 |
| 建议阶段 | Growth 起即可使用 | Mature 阶段，月广告预算 > $15K |

### SD 策略（所有卖家适用）

```
SD Retargeting（再营销）
├── 浏览过本 ASIN 未购买 → 竞价中等，频率 7 天
├── 浏览过竞品 ASIN → 竞价中高，截流
└── 浏览过本品类 → 竞价低，扩量

SD Audience（受众定向）
├── In-market 人群 → 近期有品类购买意图
└── Lifestyle 人群 → 长期兴趣标签
```

### DSP 再营销漏斗

仅在满足以下条件时建议启用 DSP：

- 月广告预算 > $15,000
- 品牌注册（Brand Registry）已完成
- 有至少 3 个月的 SP/SD 数据积累

DSP 漏斗设计：

```
Awareness（认知）    → In-market + Lifestyle 人群，CPM 竞价
  ↓
Consideration（考虑）→ 浏览过品类页/竞品页的用户，CPC 竞价
  ↓
Purchase（购买）     → 加购未购买 / 浏览本 ASIN 未购买，CPC 竞价
  ↓
Loyalty（忠诚）      → 已购买用户，复购/交叉销售，CPM 竞价
```

ROI 预期：DSP 的直接 ROAS 通常低于 SP（2-4x vs 4-8x），但其价值在于扩大漏斗顶部。评估 DSP 效果看 **New-to-Brand 比例**和 **总销量增长**，不单看 DSP ROAS。

## Module 3: Google Ads 策略（独立站 / 品牌词）

> 适用场景：有独立站的跨境卖家，或需要品牌词防御的 Amazon 卖家。

### 3.1 Search Campaign — 品牌词防御

目的：防止竞品抢占你的品牌词搜索结果。

```
Campaign: Brand Defense
├── Ad Group: Brand Exact
│   ├── [品牌名]
│   ├── [品牌名] + official
│   └── [品牌名] + website
├── Ad Group: Brand + Product
│   ├── [品牌名] + [产品名]
│   └── [品牌名] + [品类]
└── 竞价策略: Target Impression Share (95%+, 页首)
```

品牌词 CPC 通常很低（$0.1-0.5），ROAS 极高（10-20x），是性价比最高的广告类型。

### 3.2 Shopping Campaign

适用于有独立站且产品 SKU > 5 的卖家：

```
Campaign 结构:
├── High Priority (低竞价) → 品类通用词流量
├── Medium Priority (中竞价) → 长尾词 + 产品属性词
└── Low Priority (高竞价) → 品牌词 + 精准产品词

Merchant Center 要求:
- Product Feed 字段完整（title, description, price, image, GTIN）
- 标题含核心关键词，前 70 字符最关键
- 图片白底，无水印，符合 Google 规范
```

### 3.3 Performance Max (PMax)

适用条件：

- 已有至少 30 天 Shopping / Search 数据
- 月预算 > $3,000（PMax 需要数据量才能学习）
- 已安装 Google Tag 且转化追踪正常

PMax 设置要点：

| 要素 | 建议 |
|------|------|
| Asset Group | 按产品线分组，每组 5+ 图片、5+ 标题、5+ 描述 |
| Audience Signal | 导入已有客户列表 + 自定义意向受众 |
| URL 扩展 | 初期关闭（Final URL expansion = Off），避免流量分散 |
| 排除词 | 通过账户级否定关键词排除品牌词（让 Brand Campaign 处理） |
| 学习期 | 前 2 周不调整，让算法学习 |

### 3.4 跨境关键词策略

| 关键词类型 | 示例 | 竞价 | 优先级 |
|-----------|------|------|--------|
| 品牌词 | `[your brand]` | 低 CPC, 高出价 | 最高 |
| 产品词 | `bamboo cutting board` | 中 CPC | 高 |
| 品类词 | `kitchen accessories` | 高 CPC | 中（需控预算） |
| 竞品词 | `[competitor] alternative` | 中高 CPC | 低（测试性投放） |
| 长尾问题词 | `best cutting board for meat` | 低 CPC | 高（转化率高） |

### 3.5 Landing Page 对齐

广告效果 50% 取决于落地页。检查项：

- 落地页标题与广告文案关键词一致（Quality Score 直接影响）
- 首屏有明确 CTA（Add to Cart / Buy Now）
- 加载速度 < 3 秒（用 PageSpeed Insights 检测）
- 移动端适配（Google 优先索引移动端）
- 有社会证明（评价、销量、媒体报道）

## Module 4: Meta Ads 策略（品牌建设 + 引流）

> 适用场景：有独立站的跨境卖家，目标是品牌建设和站外引流。Amazon-only 卖家通常不需要本模块。

### 4.1 Campaign 架构（三阶段漏斗）

```
Awareness（认知层）— 目标：触达新用户
├── 兴趣受众（Interest-based）
├── Lookalike 1-3%（基于已购客户）
└── 素材：品牌故事视频、产品使用场景、UGC 内容
    竞价目标：CPM，预算占比 20-30%

Consideration（考虑层）— 目标：引导互动
├── 互动过广告/主页的用户
├── 网站访客（Pixel 数据）
├── 视频观看 50%+ 的用户
└── 素材：产品对比、用户评价、开箱视频
    竞价目标：Landing Page Views / Add to Cart，预算占比 30-40%

Conversion（转化层）— 目标：促成购买
├── 加购未购买用户（7天内）
├── 网站访客（3天内）
├── 已购客户（交叉销售/复购）
└── 素材：限时优惠、库存紧迫、好评截图
    竞价目标：Purchase，预算占比 30-40%
```

### 4.2 受众构建

| 受众类型 | 构建方式 | 适用阶段 | 预期 CPM |
|---------|---------|---------|---------|
| Broad（宽泛） | 仅限年龄 + 地区，让算法学习 | 预算充足时测试 | $8-15 |
| Interest（兴趣） | 品类相关兴趣标签组合 | Awareness | $10-20 |
| Lookalike 1% | 基于已购客户种子 | Consideration | $12-25 |
| Lookalike 3-5% | 扩大覆盖 | Awareness 扩量 | $8-18 |
| Custom Audience | 网站访客、邮件列表、互动用户 | Retargeting | $15-30 |
| 已购客户 | 客户列表上传 | 排除 + 交叉销售 | $10-20 |

种子受众最低要求：1,000 人（建议 5,000+ 效果更好）。

### 4.3 素材策略

跨境产品素材制作优先级：

| 优先级 | 素材类型 | 制作成本 | 效果排名 |
|--------|---------|---------|---------|
| P0 | UGC 风格短视频（15-30s） | 低 | 最高 |
| P1 | 产品使用场景图（Lifestyle） | 中 | 高 |
| P2 | 产品对比图/信息图 | 低 | 中高 |
| P3 | 开箱/评测视频（60s+） | 中 | 中 |
| P4 | 品牌故事视频 | 高 | 中（长期价值） |

素材轮换规则：

- 每个 Ad Set 放 3-5 个素材
- CTR 下降 20% 或 CPM 上升 30% → 素材疲劳，替换
- 每 2 周上新 1-2 个素材
- 表现好的素材做微调变体（换文案/换 CTA/换前 3 秒）

### 4.4 预算分配框架

月预算 < $3,000（测试期）：

```
Conversion（转化层）: 60%  — 优先验证产品是否跑得通
Consideration（考虑层）: 30%  — 积累 Pixel 数据
Awareness（认知层）: 10%  — 少量测试
```

月预算 $3,000-$10,000（增长期）：

```
Conversion: 40%
Consideration: 35%
Awareness: 25%
```

月预算 > $10,000（规模化）：

```
Conversion: 30%
Consideration: 30%
Awareness: 40%  — 持续扩大漏斗顶部
```

### 4.5 Pixel & 转化追踪

上线前必须完成：

1. **Facebook Pixel** 安装并验证（用 Facebook Pixel Helper 插件检查）
2. **Conversions API (CAPI)** 服务端追踪（弥补 iOS 14+ 数据丢失）
3. **标准事件配置**：ViewContent → AddToCart → InitiateCheckout → Purchase
4. **域名验证** + **聚合事件衡量** (AEM) 配置
5. **UTM 参数**：所有广告链接加 `utm_source=facebook&utm_medium=paid&utm_campaign={campaign_name}`

> 没有完善的转化追踪，Meta Ads 的算法优化等于失明。这是所有投放的前提。

## Module 5: 预算分配与优化框架

### 5.1 跨平台预算分配

根据业务模式选择分配方案：

**方案 A: Amazon-only 卖家**

| 渠道 | 预算占比 | 说明 |
|------|---------|------|
| Amazon SP (Sponsored Products) | 60-70% | 核心转化引擎 |
| Amazon SB/SBV (Sponsored Brands) | 15-20% | 品牌曝光 + 品类截流 |
| Amazon SD (Sponsored Display) | 10-15% | 再营销 + 竞品截流 |
| Amazon DSP | 0-10% | 月预算 > $15K 时考虑 |

**方案 B: Amazon + 独立站**

| 渠道 | 预算占比 | 说明 |
|------|---------|------|
| Amazon PPC 总计 | 50-60% | 主力销售渠道 |
| Google Ads | 20-25% | 品牌词 + Shopping + PMax |
| Meta Ads | 15-25% | 品牌建设 + 独立站引流 |

**方案 C: 独立站为主**

| 渠道 | 预算占比 | 说明 |
|------|---------|------|
| Meta Ads | 40-50% | 主力获客渠道 |
| Google Ads | 30-40% | 搜索意图捕获 |
| Amazon PPC | 10-20% | 辅助渠道（如有 Amazon 店铺） |

### 5.2 产品生命周期 × 广告策略映射

```
Launch（0-3月）
├── 目标：排名 + 评论积累
├── 预算强度：激进（广告占销售额 30-50%）
├── 重点渠道：Amazon SP Auto + Broad
├── ACOS 容忍度：高（50-80%）
└── 关键动作：大量关键词测试，快速否定无效词

Growth（3-6月）
├── 目标：扩大市场份额
├── 预算强度：中高（广告占销售额 20-30%）
├── 重点渠道：SP Exact + Phrase + SB
├── ACOS 目标：30-50%
└── 关键动作：聚焦高转化词，开启 SD 再营销

Mature（6-12月）
├── 目标：利润最大化
├── 预算强度：中等（广告占销售额 10-20%）
├── 重点渠道：SP Exact + SB + SD
├── ACOS 目标：15-30%
└── 关键动作：优化竞价，扩展 Google/Meta（如有独立站）

Harvest（12月+）
├── 目标：利润收割
├── 预算强度：保守（广告占销售额 5-10%）
├── 重点渠道：品牌词 Exact + SB 防御
├── ACOS 目标：10-20%
└── 关键动作：减少品类通用词花费，聚焦品牌词
```

### 5.3 核心指标仪表盘

| 指标 | 计算公式 | 健康范围 | 异常信号 |
|------|---------|---------|---------|
| ACOS | 广告花费 / 广告销售额 | 按阶段见上表 | 持续高于目标值 1.5x |
| TACOS | 广告花费 / 总销售额 | 8-15%（成熟期） | > 25% 说明过度依赖广告 |
| ROAS | 广告销售额 / 广告花费 | 3-8x（因品类而异） | < 2x 需立即优化 |
| CPC | 单次点击成本 | 品类基准 ±30% | 突然上升 50% 需排查 |
| CTR | 点击 / 曝光 | SP: 0.3-1.0%, SB: 0.5-1.5% | < 0.2% 检查素材和关键词 |
| CVR | 订单 / 点击 | 8-15%（Amazon） | < 5% 检查 Listing 和定价 |

### 5.4 优化节奏

| 频率 | 操作 |
|------|------|
| 每日 | 检查预算是否花完（Budget Capped）、异常花费 |
| 每周 | 搜索词报告 → 否定词 + 新词迁移、竞价调整 |
| 每两周 | Campaign 级 ACOS/ROAS 检视、预算重新分配 |
| 每月 | 全渠道 ROI 分析、生命周期阶段评估、策略调整 |
| 每季度 | 竞争格局复盘、新渠道测试决策 |

### 5.5 扩量与缩量规则

**扩量信号（增加预算）：**

- ACOS 连续 2 周低于目标值 20%+
- Campaign 每日 Budget Capped（预算花完）
- 关键词排名稳定在首页
- 转化率稳定或上升

扩量方式：每次增加 15-25%，等 3-5 天观察数据后再决定下一步。

**缩量信号（减少预算）：**

- ACOS 连续 2 周高于目标值 50%+
- CVR 持续下降且 Listing 无变化
- 库存不足（广告推起来没货更危险）
- 竞争激烈期 CPC 暴涨（如 Prime Day 前 1 周非必要不抢流量）

缩量方式：先降竞价 15-20%，观察 3 天；若无改善再砍预算。

## Module 6: 广告诊断（现有投放优化）

> 当用户提供了现有广告数据时，优先进入本模块。

### 6.1 ACOS / ROAS 健康检查

诊断流程：

```
Step 1: 确认产品阶段和利润率
  → 计算盈亏平衡 ACOS = 利润率（如利润率 30%，则 ACOS > 30% 即亏损）

Step 2: 对比实际 ACOS vs 目标 ACOS
  → 差距 < 20%: 微调优化
  → 差距 20-50%: 结构性调整
  → 差距 > 50%: 需要重建 Campaign

Step 3: 拆解 ACOS 恶化原因
  ACOS 高 = CPC 高 × CVR 低
  ├── CPC 高 → 竞价过高 / 关键词竞争激烈 / 广告相关性差
  └── CVR 低 → Listing 质量差 / 定价不合理 / 评价不足 / 关键词不精准
```

### 6.2 浪费花费识别

检查以下常见浪费场景：

| 浪费类型 | 判断标准 | 止损操作 |
|---------|---------|---------|
| 无转化关键词 | 花费 > 3x 产品售价，0 订单 | 立即暂停 + 加否定 |
| 低效 Auto Campaign | Auto ACOS > 手动 ACOS 2 倍 | 降低 Auto 预算，迁移好词到手动 |
| 品类大词烧钱 | 单个大词占总花费 > 40%，ACOS 不达标 | 降竞价或暂停，聚焦长尾 |
| 无效 ASIN 定向 | SD 定向竞品 ASIN 无转化 | 暂停该 ASIN 定向 |
| 时段浪费 | 凌晨 0-6 点花费占比 > 20% 但转化极低 | 开启分时竞价调整 |

### 6.3 关键词 / 受众优化建议

```
关键词健康度分级：
  🟢 A 级词：ACOS < 目标值，有稳定转化 → 提高竞价，确保曝光
  🟡 B 级词：ACOS 在目标值附近，有转化 → 维持观察
  🟠 C 级词：ACOS 高于目标值 50%，少量转化 → 降低竞价 20%
  🔴 D 级词：花费高，无转化或 ACOS > 目标值 2 倍 → 暂停

受众优化（Meta/Google）：
  - 合并表现相似的 Ad Set，避免自我竞争
  - 排除已转化用户（除非做交叉销售）
  - Lookalike 效果下降时更新种子受众
```

### 6.4 素材疲劳检测

| 信号 | 阈值 | 应对 |
|------|------|------|
| CTR 下降 | 较峰值下降 > 30% | 更换素材 |
| CPM 上升 | 较基准上升 > 40% | 更换素材 + 调整受众 |
| Frequency | > 3.0（Conversion 层）/ > 2.0（Awareness 层） | 扩大受众或换素材 |
| 同一素材投放时长 | > 3 周无调整 | 至少做微调变体 |

### 6.5 预算再分配建议

诊断完成后，输出预算再分配方案：

```
当前分配 vs 建议分配

| Campaign / 渠道 | 当前预算 | 当前 ROAS | 建议预算 | 调整原因 |
|----------------|---------|----------|---------|---------|
| SP Exact       | $X      | X.Xx     | $Y      | ...     |
| SP Broad       | $X      | X.Xx     | $Y      | ...     |
| ...            | ...     | ...      | ...     | ...     |
```

原则：把钱从低效 Campaign 转移到高效 Campaign，而不是简单砍预算。

## Module 7: 防御性投放策略

> 2026 年跨境广告成本持续上涨，CPC 同比涨幅 15-30%。防御性投放的核心不是"少花钱"，而是"花同样的钱守住更多利润"。

### 7.1 品牌词防御

竞品劫持品牌词是最常见的攻击方式。品牌词 CPC 低但一旦丢失，转化直接被截走。

**防御矩阵：**

| 防御手段 | 实施方式 | 预算占比建议 | 优先级 |
|---------|---------|-------------|-------|
| SP Exact 品牌词 | 所有品牌词（含拼写变体）Exact Match，竞价高于建议 30% | 5-8% 总预算 | 🔴 必须 |
| SB 品牌旗舰 | Sponsored Brands 品牌词触发，占据搜索结果顶部 | 3-5% 总预算 | 🔴 必须 |
| SD 品牌 ASIN 防守 | 在自己 ASIN 详情页投放 SD，阻止竞品出现 | 2-3% 总预算 | 🟡 强烈建议 |
| Brand Registry + Brand Story | 利用 A+ Brand Story 占据更多详情页空间 | 无额外广告费 | 🔴 必须 |

**品牌词防御检查：**
```
1. 搜索自己的品牌词 → 首条结果是否是自己？
2. 自己的 ASIN 详情页 → "Sponsored related to" 是否有竞品？
3. 品牌词的自然排名 → 是否稳定在第一位？
4. 品牌分析（Brand Analytics）→ 品牌词搜索份额是否下降？
```

### 7.2 核心 ASIN 护城河

对于贡献 80% 收入的核心 ASIN，建立广告护城河：

**ASIN 防御层：**

```
Layer 1: 搜索结果防御
├── 核心关键词 Top-of-Search 竞价加成 50-100%
├── 使用 Placement Modifier 确保首屏曝光
└── 长尾关键词矩阵覆盖（50+ 精准长尾词）

Layer 2: 详情页防御
├── SD Product Targeting 定向自己的 ASIN
├── 在互补 ASIN 上投放（自己产品的配件/耗材）
└── 竞品详情页拦截（在竞品页面出现自己广告）

Layer 3: 再营销防御
├── SD Remarketing 看过未买的用户
├── DSP Retargeting 加购未下单用户
└── Subscribe & Save 提升复购锁定
```

### 7.3 竞品攻击识别与反制

**攻击信号识别：**

| 信号 | 判断标准 | 可能原因 |
|------|---------|---------|
| 品牌词 CPC 突然上涨 > 50% | 近 7 天 vs 前 30 天均值 | 竞品抢品牌词 |
| 核心词自然排名下降 | 从首页掉到第二页 | 竞品大量广告抢位 |
| 广告 CVR 下降但流量不变 | CVR 降 > 20%，Impression 稳定 | 竞品价格战或差评攻击 |
| 搜索份额被侵蚀 | Brand Analytics 品牌搜索份额周环比降 > 10% | 竞品多维度进攻 |
| 竞品在你的详情页频繁出现 | 手动检查或用工具监控 | 竞品 SD 定向攻击 |

**反制策略：**

```
情境 A: 竞品抢品牌词
  → 提高品牌词竞价 + 开 SB Video 品牌词 + 联系 Amazon Brand Registry 投诉

情境 B: 竞品在你的详情页投广告
  → SD 自我定向（在自己页面投自己广告） + 提升 A+ Content 说服力

情境 C: 竞品价格战导致 CVR 下降
  → 不跟价！用 Coupon 替代降价（不影响长期价格锚点） + 强化差异化卖点

情境 D: 竞品大词砸钱抢排名
  → 避开大词正面战，迂回长尾词矩阵 + 聚焦 CVR 提升而非 CPC 对抗
```

### 7.4 降本增效组合拳

在广告成本上涨环境下的系统性降本策略：

**降 CPC 策略：**

| 策略 | 预期效果 | 实施难度 |
|------|---------|---------|
| 长尾关键词矩阵（50-100 个低竞争词） | CPC 降 30-50% | ⭐⭐ |
| 分时竞价（高转化时段加价，低转化时段降价） | 整体 CPC 降 10-20% | ⭐⭐⭐ |
| 否定关键词体系（周频清理 Search Term Report） | CPC 降 5-15% + 减少浪费 | ⭐⭐ |
| 产品定向替代关键词定向（部分场景） | CPC 降 20-40% | ⭐⭐ |
| 新 Match Type 测试（Broad + Modifier 替代 Phrase） | CPC 变化不定，扩大覆盖 | ⭐⭐ |

**提 CVR 策略（同样 CPC 下降 ACOS）：**

| 策略 | 预期效果 | 需要配合 |
|------|---------|---------|
| 主图优化（A/B Test） | CVR 提升 10-30% | mc-listing |
| 价格锚点调整（划线价 + Coupon 组合） | CVR 提升 5-15% | mc-cost |
| Review 数量/评分提升 | CVR 提升 10-25% | 运营 |
| A+ Content 升级（Premium A+） | CVR 提升 5-15% | mc-listing |
| Video 广告替代图片广告 | CTR 提升 20-40% | 素材制作 |

### 7.5 防御性预算分配模型

```
总广告预算 = 进攻预算 + 防御预算 + 测试预算

推荐比例（按产品生命周期）：

新品期（0-3 月）：
  进攻 70% · 防御 15% · 测试 15%
  → 重点抢排名，防御仅做品牌词基础防护

成长期（3-12 月）：
  进攻 50% · 防御 30% · 测试 20%
  → 开始建立 ASIN 护城河，测试新渠道

成熟期（12 月+）：
  进攻 30% · 防御 50% · 测试 20%
  → 防御为主，守住排名和利润率

衰退期 / 清仓期：
  进攻 10% · 防御 20% · 测试 0% · 削减 70%
  → 仅保留品牌词和核心 ASIN 防御
```

## 交互规则

- 用户提供现有广告数据时，优先进入 Module 6 诊断模式
- 用户只做 Amazon 时，跳过 Module 3 和 Module 4
- 用户只做独立站时，跳过 Module 1 和 Module 2
- 用户提到"品牌词被抢"或"竞品攻击"时，优先进入 Module 7 防御模式
- 用户问具体问题（如"ACOS 太高怎么办"）时，直接给出针对性建议，不需要跑完全部模块
- 所有数值建议均标注"品类差异较大，需用实际数据校准"
