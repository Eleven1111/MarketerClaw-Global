## 执行步骤

### Step 1: 了解团队现状

收集用户的团队规模、运营平台、当前痛点。如未提供，追问：
- 你的团队多少人？各自负责什么？
- 运营哪些平台？哪个是主力？
- 最大的效率瓶颈在哪？

### Step 2: 选择模块

根据用户需求选择对应模块执行：
- "日常运营怎么安排" → Module 1
- "多平台怎么管" → Module 2
- "帮我做个复盘模板" → Module 3
- "新品上架流程" → Module 4
- "团队怎么分工" → Module 5
- "给我一个运营日历" → Module 6
- "全部都要" → 按 Module 1→6 顺序执行

### Step 3: 定制化输出

根据用户的平台、团队规模、具体产品定制 SOP 内容。不要输出与用户无关的平台内容。

### Step 4: 生成可执行文档

输出格式化的 SOP / 复盘模板 / 日历，用户可以直接复制使用。

### Step 5: 关联其他技能

在 SOP 中标注与其他技能的衔接点：
- 选品环节 → mc-selection
- 成本计算 → mc-cost
- Listing 优化 → mc-listing
- 广告策略 → mc-ads
- 合规检查 → mc-compliance
- 竞品分析 → mc-compete / mc-monitor
- 物流决策 → mc-logistics

### Step 6: 交付

按交付规范输出摘要卡片。

## 常见陷阱

1. **SOP 写了不用** — 最好的 SOP 是最简单的。一页纸 > 十页文档。贴在工位上比存在网盘里有用。

2. **所有平台同等投入** — 新手最常犯的错。3 个平台各投 33% 精力 = 3 个平台都做不好。先把一个平台做到稳定盈利再扩。

3. **不做数据复盘** — "感觉还行"是最危险的。每周 2 小时复盘，比每天多干 2 小时更有效。

4. **新品不设 Kill Line** — 沉没成本谬误。设定明确的止损标准，到了就执行，不要"再给它一次机会"。

5. **团队没有 SOP 就招人** — 新人来了不知道干什么，老板天天教 = 更忙。先写 SOP，再招人。

6. **忽略时区差异** — 中国晚上 8-10 点是美国白天高峰。如果这个时段不做广告调整，会错过最佳优化窗口。

7. **只看收入不看利润** — 月销 $50K 但净利 $2K 的产品不如月销 $10K 净利 $3K 的产品。用 mc-cost 定期审计每个 SKU 的真实利润。

## 核心问题

- 我们的运营时间花在最有杠杆的事情上了吗？
- 团队中最大的瓶颈岗位是什么？
- 如果明天有人离职，其他人能接手吗？
- 我们的复盘是否驱动了实际的运营改进？
- 新品成功率是多少？失败产品有没有共性？
- 库存断货和积压的频率是否在下降？
- 我们的 SOP 上次更新是什么时候？还适用吗？
- 每个人是否清楚自己的 KPI 和考核标准？

## 交互规则

- 用户可以只要某个模块（如"只给我复盘模板" / "新品上架 checklist"）
- 根据用户的平台组合裁剪内容，不输出无关平台的 SOP
- 根据团队规模调整复杂度（Solo 要简单，15 人团队要精细）
- SOP 模板中的 {变量} 用用户提供的实际信息填充
- 复盘模板中的 KPI 目标值根据用户的品类和阶段调整
