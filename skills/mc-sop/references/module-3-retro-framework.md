### 3.3 季度业务回顾 (QBR) 框架

```
一、品类表现排名
  · 各品类收入贡献占比
  · 各品类利润贡献占比（收入高不一定利润高）
  · 品类健康度：增长型 / 稳定型 / 衰退型

二、新品成功率
  · 本季度上新数量：{N}
  · 达标产品（月销 > {X} 单）：{N}（成功率 {X}%）
  · 失败产品分析：为什么没起来？共性问题？

三、YoY 增长分析
  · 收入 YoY：{X}%
  · 利润 YoY：{X}%
  · SKU 数量变化
  · 市场份额变化

四、战略复盘
  · 上季度战略目标完成情况
  · 最大的成功是什么？可否复制？
  · 最大的失误是什么？教训是什么？
  · 下季度战略方向调整
```

### 3.4 复盘 → Brand Brain 记忆回写（复盘后必做）

周度/月度/季度复盘输出完成后，把本周期**验证或证伪的假设**蒸馏进品牌长期记忆，
让洞察跨周期复利——这是复盘的最后一步，不是可选项：

1. 从复盘结论中提取 ≤3 条**已被数据验证**的洞察
   （finding 格式：`{结论} — 验证于 {周期/产品}，证据：{数据点}`）
2. 提取 ≤2 条**被证伪的假设**（策略判断被实际数据推翻的）
   （finding 格式：`{原假设} — 证伪于 {周期/产品}，实际：{数据点}`）
3. 按 CLAUDE.md 的 Learnings 日志规范追加到 `brand-brain/learnings.jsonl`：
   验证的记 `"status":"verified"`，证伪的记 `"status":"falsified"`；
   只追加不重写，与既有条目语义重复的跳过
4. 无执行数据的预测性复盘**不回写**（未经验证的判断不能进入长期记忆）

回写完成后在交付卡中加一行：`🧠 品牌记忆：+{N} 条验证洞察 / +{M} 条证伪记录`
